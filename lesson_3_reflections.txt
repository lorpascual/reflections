When would you want to use a remote repository rather than keeping all your work local?
When I'm part of a development team and I need to share my changes to the repository.

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?
When you pull changes you are aware of what are you exactly pulling and when you want to do it. However you take
the risk of forget to pull and important change and later have to make a huge pull.

Describe the differences between forks, clones, and branches. When would you use one instead of another?
Fork is a copy of a git hub repo that is only on git hub, no locally. 
You make a clone when you make a copy of a git hub repository to your local computer.
Branches are ramifications of a repository no matter if is local or on Git Hub.
